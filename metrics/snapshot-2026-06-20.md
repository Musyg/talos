# Platform metrics snapshot - 2026-06-20

Every headline figure in the project overview is measured on the running fleet,
read-only, not estimated. This snapshot records how each number was obtained so it
can be reproduced. Node roles are referenced generically (one control/business
node, two GPU worker nodes, one support node); host names, network addresses, and
security-pillar data are intentionally omitted. All probes are read-only: no
writes, no restarts.

## Active platform agents: 54 across the fleet

manager:

```
systemctl list-units 'hermes*' --type=service --state=running --no-legend | wc -l
```

| Node | Running agent-services |
| --- | --- |
| Control / business node | 30 |
| GPU worker A | 9 |
| GPU worker B | 12 |
| Linux fleet subtotal | 51 |

The support node (a Windows workstation) runs 3 additional control and scraping
services, counted from its process and port list, for a fleet total of 54.

Shared infrastructure runs in addition to the agents above: the event bus, the
vector database, time-series metrics, containerized services, and the model
gateways.
